# 🗂 design-notes フォルダについて

このフォルダは、「設計仕様書」ほど厳密ではないが、「dev-thoughts」ほどカジュアルでもない、
**中間層の設計ノート**を記録する場所です。

## 📌 内容の特徴
- 抽象的な構想、設計方針、要素分類、概念整理
- 実装を進める上での設計的背景や統一基準
- 他ファイルにまたがる再利用的アイデアのまとめ

## 📎このフォルダに含まれるノート
- `FSM_Structure.md` : FSMの構造、スコープ、副作用分類などの設計メモ


## 🧭 関連ドキュメント
- `TODO.md` : 実装タスクと進捗
- `dev-thoughts/` : 思考のフリーログ、直感や疑問
